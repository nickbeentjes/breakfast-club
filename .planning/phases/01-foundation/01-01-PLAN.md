**src/schema/identity.schema.json** â€” JSON Schema (draft 2020-12) for the identity document. The schema validates the full nick-identity.json structure. Key design points:

- Top-level required fields: `schema_version` (string, semver pattern), `identity_created` (string, ISO 8601), `source` (string)
- Four sections, each required: `persona`, `skills`, `projects`, `values`
- Each section carries a `_sensitivity` field (enum: `"public"`, `"professional"`, `"personal"`, `"private"`) for Phase 2 projection enforcement (IDNT-05)
- `persona._sensitivity` defaults to `"personal"` (contains location, relationships)
- `skills._sensitivity` defaults to `"professional"`
- `projects._sensitivity` defaults to `"professional"`
- `values._sensitivity` defaults to `"personal"`
- `persona` object: required `name` (string), `communication_style` (object with `primary`, `preferences` array, `dislikes` array), `working_style` (object with `approach`, `tools`, `strengths` array). Optional: `location`, `input_method`.
- `skills` object: required `primary_stack` (object), `domain_expertise` (object with patternProperties for domain entries each having `depth` enum and `details` string). Optional: `trades_knowledge` (array of strings).
- `projects` object: required `active` (array of objects each with `name`, `type`, `status`, `description`; optional `stack`, `domain`, `location`). Optional: `completed` (array).
- `values` object: required `professional` (array of strings). Optional: `technical_opinions` (array of strings).
- Use `additionalProperties: true` at section level to allow future fields without schema breakage.

**src/types.ts** â€” TypeScript interfaces mirroring the schema and the MongoDB document shape from RESEARCH.md:

```typescript
export type SensitivityLevel = "public" | "professional" | "personal" | "private";

export type IdentitySection = "persona" | "skills" | "projects" | "values";

export interface IdentityDocument {
  _id?: import("mongodb").ObjectId;
  doc_type: "identity";
  section: IdentitySection;
  sensitivity: SensitivityLevel;
  schema_version: string;
  content: Record<string, unknown>;
  embedding: number[];
  embedding_model: string;
  source_hash?: string;
  git_tree_sha?: string;
  updated_at: Date;
}

export interface IdentitySchema {
  schema_version: string;
  identity_created: string;
  source: string;
  persona: PersonaSection;
  skills: SkillsSection;
  projects: ProjectsSection;
  values: ValuesSection;
}
```

Also export sub-interfaces for PersonaSection, SkillsSection, ProjectsSection, ValuesSection with their respective fields.
