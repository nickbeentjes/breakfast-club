    **src/projection/apply-projection.ts** — Create the whitelist filter function per RESEARCH.md Pattern 1. Import `IdentityDocument` from `../types.js` and `ProjectionDefinition` from `./types.js`. The function signature:

    ```typescript
    export function applyProjection(
      docs: IdentityDocument[],
      projection: ProjectionDefinition | null | undefined
    ): IdentityDocument[]
    ```

    Critical invariant: if `projection` is null/undefined, return `[]` — NEVER return the full documents. This is the "fail closed" safety contract (per PROJ-01).

    Implementation:
    1. If `!projection`, return `[]`
    2. Filter docs by `projection.allowed_sections.includes(doc.section)`
    3. Filter docs by `projection.allowed_sensitivity.includes(doc.sensitivity)`
    4. Map remaining docs: if `projection.field_allowlist?.[doc.section]` exists, filter `doc.content` keys to only those in the allowlist. Otherwise return full content.
    5. Return a new array (never mutate input docs). Use spread on doc to create shallow copy with filtered content.

    Helper function `filterContent(content: Record<string, unknown>, allowlist?: string[]): Record<string, unknown>` — if no allowlist, return content as-is; otherwise `Object.fromEntries(Object.entries(content).filter(([key]) => allowlist.includes(key)))`.

    **src/projection/load-projections.ts:**

    Import `readFileSync`, `readdirSync`, `existsSync` from `node:fs`, `join` from `node:path`, `z` from `zod`, and `ProjectionDefinition` from `./types.js`.

    Define a Zod schema for projection validation:
    ```typescript
    const projectionSchema = z.object({
      name: z.string().min(1),
      description: z.string(),
      allowed_sections: z.array(z.enum(["persona", "skills", "projects", "values"])).min(1),
      allowed_sensitivity: z.array(z.enum(["public", "professional", "personal", "private"])).min(1),
      field_allowlist: z.record(z.string(), z.array(z.string())).optional(),
    });
    ```

    Export function `loadProjections(projectionsDir: string): Map<string, ProjectionDefinition>`:
    1. `readdirSync(projectionsDir)` filtering for `.json` files
    2. For each file: `readFileSync` + `JSON.parse` + `projectionSchema.parse()`
    3. If parse fails, throw with clear error message including filename
    4. Add to Map keyed by `parsed.name`
    5. Return the Map

    Also export `getProjection(projections: Map<string, ProjectionDefinition>, name: string): ProjectionDefinition | undefined` — simple `Map.get()` wrapper for clarity.
